
# Mobile App Generation & Sync Framework (React Native/Expo)

## 1. Introduction

Extending the AI MVP Builder to generate robust mobile applications is crucial for a comprehensive "Base44-like" experience. This framework outlines the strategy for the AI to generate cross-platform mobile applications using technologies like React Native/Expo or Flutter, ensuring seamless synchronization with the AI-generated backend and maintaining the "vibe coding" instant feedback loop. The goal is to enable users to describe a mobile app, and have the AI generate a functional, deployed application that shares the same data and business logic as its web counterpart.

## 2. Mobile Code Generation Strategy: The Developer Agent for Mobile

The Developer Agent, already proficient in web frontend generation, will be enhanced to generate mobile application code. The choice between React Native/Expo and Flutter will depend on user preference or a default setting within the platform.

### 2.1. React Native/Expo Generation
-   **Process**: When a user specifies a mobile application or a cross-platform application, the Developer Agent will generate React Native code, often leveraging the Expo framework for simplified development and deployment.
-   **Component Reusability**: The AI will be trained to identify common UI patterns and generate reusable mobile components (e.g., `FlatList` for lists, `TextInput` for forms) that align with the application's data model.
-   **Styling**: Utilize styling solutions like `StyleSheet` or libraries like `NativeBase`/`Tamagui` for consistent theming and responsive design across different mobile devices.
-   **Navigation**: Generate navigation structures using `React Navigation` to define screens and their flow.
-   **Output**: Modular React Native/Expo code (e.g., `src/screens/HomeScreen.tsx`, `src/components/MobileDealCard.tsx`).

### 2.2. Flutter Generation (Alternative)
-   **Process**: If Flutter is chosen, the Developer Agent will generate Dart code for Flutter widgets and screens. This involves understanding Flutter's declarative UI paradigm.
-   **Widget Tree Construction**: The AI will build widget trees that represent the mobile UI, ensuring proper layout and state management.
-   **State Management**: Generate code using Flutter's state management solutions (e.g., Provider, Riverpod, BLoC) to handle application state.
-   **Output**: Modular Flutter code (e.g., `lib/screens/home_screen.dart`, `lib/widgets/deal_card.dart`).

### 2.3. Shared Business Logic and API Integration
-   **Principle**: The mobile application, regardless of its framework, must consume the same backend API and adhere to the same data models defined by the Architect Agent for the web application.
-   **Process**: The Developer Agent will generate mobile client code that:
    -   **Uses the same API Endpoints**: Fetches and sends data to the backend API generated by the Developer Agent for the web app.
    -   **Shares Data Models**: Utilizes TypeScript interfaces (for React Native) or Dart classes (for Flutter) that mirror the PostgreSQL schema and API response structures.
    -   **Authentication**: Integrates with Supabase Auth using mobile SDKs, ensuring a consistent authentication experience across web and mobile.

## 3. Cross-Platform Consistency and Data Synchronization

Maintaining consistency between web and mobile applications is paramount for a unified user experience and simplified development.

### 3.1. Unified Data Model
-   **Strategy**: The Architect Agent's generated PostgreSQL schema (managed by Supabase) serves as the single source of truth for data. Both web and mobile applications will derive their data structures from this schema.
-   **Type Generation**: Automated generation of TypeScript types (for React/React Native) or Dart classes (for Flutter) directly from the Supabase schema ensures type safety and consistency across the entire stack.

### 3.2. Real-time Data Synchronization for Mobile
-   **Challenge**: Mobile applications often require real-time updates to provide a dynamic user experience (e.g., live chat, collaborative features, real-time dashboards).
-   **Solution**: Leverage **Supabase Realtime** for instant data synchronization.
    -   **Process**: The Developer Agent will generate mobile client code that subscribes to changes in the Supabase database. When data is inserted, updated, or deleted on the backend (either by the web app or another mobile client), the mobile app's UI will automatically reflect these changes.
    -   **Implementation**: Use Supabase's client-side SDKs (JavaScript for React Native, Dart for Flutter) to listen for database changes on specific tables or rows.

### 3.3. Offline Capabilities (Future Enhancement)
-   **Strategy**: For enhanced mobile experience, especially in areas with poor connectivity, the AI could generate code for offline-first capabilities.
-   **Implementation**: Utilize local storage solutions (e.g., `AsyncStorage` for React Native, `Hive`/`sqflite` for Flutter) and synchronization logic to allow users to interact with data offline and sync when connectivity is restored.

## 4. Mobile Preview and Deployment

Extending the "vibe coding" experience to mobile requires specialized preview and deployment mechanisms.

### 4.1. Instant Mobile Preview
-   **Challenge**: Running mobile applications directly in a browser (like WebContainers for web) is not always feasible for native features.
-   **Solution**: The AI MVP Builder will offer multiple preview options:
    -   **Expo Go/Expo Development Client**: For React Native/Expo apps, the Developer Agent will generate a QR code that users can scan with the Expo Go app on their physical device or an Expo Development Client. This provides a near-native live preview.
    -   **Web-based Simulators**: For basic UI, a web-based simulator (e.g., provided by Expo Snack or similar services) can offer a quick, albeit limited, preview directly in the browser.
    -   **Hot Reloading**: Similar to web development, changes generated by the AI will trigger hot reloading in the Expo Go app or simulator, providing instant visual feedback.

### 4.2. Automated Mobile Deployment
-   **Process**: The DevOps Agent will automate the build and deployment process for mobile applications.
    -   **App Store Submission**: For production-ready apps, the DevOps Agent will generate configurations and scripts for automated builds and submissions to Apple App Store (TestFlight) and Google Play Store.
    -   **Over-the-Air (OTA) Updates**: For Expo projects, the DevOps Agent will configure OTA updates, allowing code changes to be deployed without requiring a full app store submission.
    -   **CI/CD for Mobile**: Integrate with mobile-specific CI/CD platforms (e.g., Expo Application Services (EAS), Bitrise, Fastlane) to automate testing, building, and distribution.

## 5. Prompt Engineering for Mobile-Specific Features

-   **Device-Specific UI**: Prompts will guide the AI to generate UI that adapts to different screen sizes and orientations, and leverages native mobile UI patterns.
    -   *Example*: "Generate a contact list screen for a mobile CRM. Use a native-looking list view with search and filter functionality. Include a floating action button to add new contacts."
-   **Native Features**: Prompts can specify the use of native device features.
    -   *Example*: "Add a feature to the mobile CRM to call a contact directly from their profile using the phone's native dialer."
-   **Performance Optimization**: Prompts can guide the AI to generate performant mobile code, considering factors like bundle size, network requests, and battery usage.
