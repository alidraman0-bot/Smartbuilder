## Quick Start

### Basic Usage via Agent
The `Base44Agent` is the primary entry point for the `MvpBuilderService`.

```python
import asyncio
from app.agents.base44_agent import Base44Agent

async def main():
    agent = Base44Agent()
    
    # Execute elite generation
    idea = "Build a task management dashboard with React and FastAPI"
    result = await agent.execute(idea)
    
    print(f"Project Overview: {result['overview']}")
    print(f"Files Generated: {len(result['files'])}")

asyncio.run(main())
```

### Direct Engine Access
For granular control over generation phases, use the `CodeGenerationEngine`.

```python
from app.core.engine import CodeGenerationEngine, GenerationPhase

async def generate_app():
    engine = CodeGenerationEngine()
    
    user_description = "Build a multi-tenant SaaS for dental clinics"
    
    # Runs all 7 phases: Requirements -> Architecture -> Schema -> Backend -> Frontend -> Config -> Docs
    results = await engine.generate_complete_app(user_description)
    
    if results['status'] == 'completed':
        for file in results['files']:
            print(f"Generated: {file['path']}")

asyncio.run(generate_app())
```

## API Reference

### GenerationPhase Enum
Located in `app/core/engine.py`:
- `REQUIREMENTS_ANALYSIS`
- `ARCHITECTURE_DESIGN`
- `DATABASE_SCHEMA`
- `BACKEND_CODE`
- `FRONTEND_CODE`
- `CONFIGURATION`
- `DOCUMENTATION`

### CodeGenerationEngine
- `generate_complete_app(user_description: str)`: Orchestrates the full 7-phase pipeline.
- `_execute_phase_call(prompt, phase)`: Internal method to call the `AIClient` for a specific phase.
- `generation_history`: List of `GenerationResult` objects for audit and troubleshooting.

### AIClient (Low-Level SDK)
The engine utilizes `app.core.ai_client.get_ai_client()` for all LLM communication.
- **Provider Switching**: Automatically respects `AI_PROVIDER` from `.env`.
- **Fallback**: Includes automatic fallback logic (Gemini -> OpenAI -> Claude) if primary providers fail.

## Advanced Features

### 1. Phased Context Injection
Each subsequent phase (e.g., Backend Code) receives the metadata generated in the Requirements phase, ensuring architectural consistency across the entire app.

### 2. "Elite" Engineering Standards
The engine enforces a strict system prompt that mandates:
- **Zero `// TODO`s**: All code must be 100% complete.
- **Type Safety**: Mandatory TypeScript with strict typing.
- **Security**: Built-in Row Level Security (RLS) for all database schemas.
- **Modularity**: Components are separated into logical service and repository layers.

### 3. Automatic Fallback
The engine is hardened against "Model Not Found" or "API Rate Limit" errors. If `gemini-1.5-pro` is unavailable, it automatically retries with `gemini-1.5-flash` or falls back to `GPT-4o`.

## Troubleshooting

### Persistence Issues
If you encounter `PGRST204` errors (Could not find 'run_id'), ensure your Supabase schema for `build_plans` is up to date.

### API Rate Limits
If all providers return `429`, the engine will log the error and transition the `BuildSession` to a stable state to prevent infinite retry loops.
